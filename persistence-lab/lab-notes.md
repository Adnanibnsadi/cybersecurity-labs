# Persistence and System Security Lab

## Overview

This lab was conducted in a controlled virtual environment to study persistence mechanisms and understand the security risks associated with unauthorized long-term access to systems.

## Lab Environment

- Host: Windows 11
- Attacker VM: Kali Linux
- Target VM: Windows 7
- Platform: VMware Workstation

## Objectives

- Understand the concept of persistence.
- Observe how long-term access can impact system security.
- Explore methods defenders can use to detect suspicious activity.
- Study the importance of endpoint security controls.

## Key Observations

- Persistence can allow access to remain available across system restarts.
- Legacy operating systems may have additional security challenges.
- Monitoring and logging are important for identifying unusual behavior.

## Defensive Considerations

- Keep systems updated and supported.
- Use endpoint protection and monitoring.
- Review startup programs, scheduled tasks, and system logs.
- Apply the principle of least privilege.
